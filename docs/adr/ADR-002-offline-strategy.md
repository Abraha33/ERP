# ADR-002 — Estrategia offline (pendiente)

## Estado

**Borrador — no aplica hasta Fase 8**

## Regla

No escribir implementación offline ni sync antes de completar el contenido de este ADR y marcarlo **Aceptado**.

El alcance mínimo y el alcance completo (lectura vs escritura, conflictos, SQLite en Expo, colas) se define aquí, no en código espontáneo.
